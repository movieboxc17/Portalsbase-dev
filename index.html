How to Host a Web Page with GitHub & Cloudflare

Step-by-step guide to deploying your site with a custom domain

1. Create a GitHub Repository

  1. Go to GitHub new repo and create a public repository (e.g. yourusername/yourwebsite).
  2. Add your website files to the repository (e.g. index.html, style.css).
  3. Commit and push your changes.

2. Enable GitHub Pages

  1. Go to your repository’s Settings > Pages.
  2. Under Source, select the branch (usually main) and folder (/root or /docs).
  3. Save. Your site will be published at https://yourusername.github.io/yourwebsite.

3. Buy or Use a Custom Domain

  1. Purchase a domain name from a registrar (Namecheap, Google Domains, etc.) or use one you already own.

4. Set Up Cloudflare for Your Domain

  1. Sign up / log in at Cloudflare.
  2. Add your domain to Cloudflare and follow the steps to change your registrar’s nameservers to Cloudflare’s.
  3. Once verified, you’ll be able to manage DNS for your domain via Cloudflare.

5. Configure DNS to Point to GitHub Pages

  1. In Cloudflare, go to your domain’s DNS settings.
  2. Add a CNAME record:
    • Name: @ or www
    • Target: yourusername.github.io
  3. For root domains (example.com), use Cloudflare’s Page Rules or configure with ALIAS/ANAME if supported, or set up an A record pointing to GitHub’s IPs:
    • A Record IPs: 185.199.108.153, 185.199.109.153, 185.199.110.153, 185.199.111.153

6. Tell GitHub Pages About Your Custom Domain

  1. In your repo, go to Settings > Pages.
  2. Under Custom domain, enter your domain (e.g. example.com).
  3. GitHub will check your DNS. Once set up, your site will be live at your custom domain.

7. (Optional) Force HTTPS

  1. In GitHub Pages settings, enable Enforce HTTPS for secure connections.
  2. Cloudflare will also automatically provide SSL for your domain.
